# 김비서 Agent 실행 가이드

---

## 개요

김비서 Agent는 일일 업무 현황을 자동으로 분석하고 경영진 브리핑을 생성하는 시스템입니다. 5개의 데이터 파일을 읽어 현재 상태를 분석하고 마크다운 형식의 전문적인 보고서를 생성합니다.

---

## 파일 구조

```
워크숍-실습/
├── 김비서-데이터/                    # 에이전트 데이터 폴더
│   ├── 매출데이터.csv               # 판매 실적 데이터
│   ├── 업무목록.csv                 # 진행 중인 업무 목록
│   ├── 주간일정.txt                 # 주간 일정 및 회의
│   ├── 프로젝트현황.csv             # 프로젝트 진행 현황
│   ├── 회의록.txt                   # 최근 회의 기록
│   └── 브리핑_*.md                  # 생성된 브리핑 (매일)
│
├── settings.json                    # 에이전트 설정 파일

└── 김비서_실행가이드.md             # 본 문서
```

## 설정 파일 설명 (settings.json)

### 주요 섹션

#### 1. agents (에이전트 정의)
```json
"김비서": {
  "name": "Kim Secretary Agent",
  "description": "일일 업무 브리핑 및 현황 관리 에이전트",
  "enabled": true,
  "config": {
    "dataFolder": "./김비서-데이터",
    "updateFrequency": "daily",
    "briefingTime": "09:00"
  }
}
```
- **enabled**: true/false로 에이전트 활성화 제어
- **dataFolder**: 데이터 파일이 있는 폴더 경로
- **updateFrequency**: 실행 주기 (daily, weekly, manual)
- **briefingTime**: 일일 브리핑 생성 시간 (HH:MM)

#### 2. dataManagement (데이터 관리)
```json
"dataPath": "./김비서-데이터",
"filesToMonitor": [
  "매출데이터.csv",
  "업무목록.csv",
  "주간일정.txt",
  "프로젝트현황.csv",
  "회의록.txt"
]
```
- 모니터링할 파일 목록
- 자동 갱신 대상 파일 지정

#### 3. briefingConfig (브리핑 설정)
```json
"schedule": "daily",
"time": "09:00",
"timezone": "Asia/Seoul",
"outputFormat": "markdown",
"outputPath": "./김비서-데이터/브리핑_{date}.md"
```
- **schedule**: daily/weekly/manual
- **time**: HH:MM 형식
- **timezone**: 시간대 (Asia/Seoul)
- **outputPath**: {date}는 YYYYMMDD로 자동 치환

#### 4. analysisRules (분석 규칙)
```json
"tasks": {
  "urgent": {
    "criteria": "마감일이 오늘 또는 내일인 업무",
    "priority": "높음"
  },
  "overdue": {
    "criteria": "마감일이 지난 업무",
    "priority": "긴급"
  }
}
```
- 업무 분류 기준
- 프로젝트 위험도 판정
- 매출 분석 범위

#### 5. hooks (자동 실행 규칙)
```json
"onDataRefresh": {
  "enabled": true,
  "actions": [
    "validateDataIntegrity",
    "parseCSVFiles",
    "parseTextFiles",
    "updateMetrics"
  ]
}
```
- 데이터 갱신 시 실행할 액션
- 브리핑 생성 시 실행할 액션

## 데이터 파일 포맷

### 1. 매출데이터.csv
```csv
날짜,제품,카테고리,수량,단가,매출액,지역
2026-01-05,무선 이어폰,전자기기,45,89000,4005000,서울
2026-01-05,보조배터리,전자기기,32,35000,1120000,서울
```
- **날짜**: YYYY-MM-DD 형식
- **카테고리**: "전자기기", "생활용품"
- **지역**: "서울", "부산", "대구"

### 2. 업무목록.csv
```csv
업무,우선순위,상태,담당자,마감일,카테고리
3월 프로모션 기획안 작성,높음,진행중,김대리,2026-03-12,마케팅
```
- **우선순위**: "높음", "보통", "낮음"
- **상태**: "진행중", "대기", "완료"
- **카테고리**: 팀/부서별 분류

### 3. 주간일정.txt
```
■ 월요일 (3/10)
- 10:00  마케팅팀 주간회의
- 14:00  신제품 런칭 킥오프 미팅
```
- 요일별 구분 (■ 표시)
- 시간별 일정 (HH:MM)
- 들여쓰기로 계층 표현

### 4. 프로젝트현황.csv
```csv
프로젝트명,진행률,상태,담당자,시작일,마감일,예산(만원),집행(만원)
봄맞이 프로모션,65,진행중,김대리,2026-02-15,2026-03-31,500,325
```
- **진행률**: 0-100 (%)
- **상태**: "준비중", "기획중", "진행중", "마무리", "완료"
- **예산/집행**: 단위는 만원

### 5. 회의록.txt
```
========================================
  마케팅팀 주간회의 - 2026년 3월 10일 (월)
========================================

■ 지난주 진행사항
- SNS 광고 캠페인 A/B 테스트 결과 나옴
```
- 제목: "========" 구분선
- 섹션: "■" 마크로 구분
- 내용: 불릿 포인트 (- 또는 *)

```markdown
# 김비서 Agent - 2026년 4월 9일(수요일) 브리핑

## 📋 오늘의 일정 및 업무
- 예약된 회의 목록

- 기간별 매출액
- 상위 제품
- 지역별 분석

## 🎯 프로젝트 현황
- 진행 중: 4개
- 마무리: 1개
- 기획 중: 1개

## 📝 진행 중인 업무
- 🔴 긴급 (D-1, D-0)
- 🟠 높음 (3일 이내)
- 🟡 보통 (1주 이내)

## 🚨 주의사항
- 종료 예정 프로젝트
- 위험 요소
- 의사결정 필요 항목

## 💡 내일(4월 10일) 예상
- 다음날 예약 일정
- 준비 필요 사항
```

### 이모지 의미
- 🔴: 긴급/필수 조치
- 🟠: 높은 우선순위
- 🟡: 일반 우선순위
- 🟢: 정상/완료
- 📋: 업무/일정
- 📊: 통계/분석
- 🎯: 프로젝트/목표
- 📝: 보고/문서
- 💡: 제안/아이디어
- ⚠️: 경고/주의
- 🚨: 긴급 상황

## 트러블슈팅

### 문제 1: 파일을 찾을 수 없음
**증상**: "File not found" 에러
**해결책**:
1. 파일 경로 확인: `settings.json`의 `dataPath` 확인
2. 인코딩 확인: 모든 파일이 UTF-8로 저장되어 있는지 확인
3. 파일명 확인: 정확한 파일명 입력 (띄어쓰기, 한글 정확성)

### 문제 2: 날짜가 잘못 해석됨
**증상**: "날짜 형식 에러" 또는 일정이 표시되지 않음
**해결책**:
1. 날짜 형식 통일: 모든 파일에서 YYYY-MM-DD 형식 사용
2. 시간대 확인: `settings.json`의 `timezone`이 "Asia/Seoul"인지 확인
3. 현재 시스템 시간 확인

### 문제 3: CSV 파싱 에러
**증상**: 데이터가 표시되지 않거나 깨짐
**해결책**:
1. CSV 구분자 확인: 쉼표(,) 사용 확인
2. 인코딩 확인: UTF-8 인코딩 확인
3. 헤더 행 확인: 첫 줄이 헤더인지 확인
4. 빈 칸 확인: 의도하지 않은 공백이 없는지 확인

### 문제 4: 브리핑이 생성되지 않음
**증상**: 출력 폴더에 파일이 생성되지 않음
**해결책**:
1. 출력 폴더 확인: `outputPath`의 폴더가 존재하는지 확인
2. 권한 확인: 쓰기 권한이 있는지 확인
3. 디스크 공간 확인: 여유 공간이 있는지 확인
4. 로그 확인: 에러 메시지 검토

## FAQ

**Q1: 브리핑을 다른 형식(PDF, Excel)으로 받을 수 있나?**  
A: 현재는 마크다운만 지원합니다. 생성된 .md 파일을 변환 도구로 PDF/Excel로 변환할 수 있습니다.

**Q2: 데이터를 수동으로 수정해도 다음 실행에 자동으로 반영되나?**  
A: 예, 파일을 저장하면 다음 브리핑 생성 시 자동으로 반영됩니다.

**Q3: 특정 날짜의 과거 브리핑을 다시 생성할 수 있나?**  
A: 가능합니다. `kim_secretary.agent --date 2026-04-01` 옵션으로 특정 날짜 지정.

**Q4: 브리핑에 특정 정보를 추가하거나 제외할 수 있나?**  
A: `settings.json`의 `includeMetrics` 섹션에서 조정 가능합니다.
